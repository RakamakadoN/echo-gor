# Аудит мобильной адаптации «эхо-гор»

Дата: 19.06.2026 · Стек: React 19 + Vite + Tailwind CSS v4 · Цель: телефоны (≤480px), планшеты (≤1024px), малые ноутбуки.

## Краткий вывод

Общая готовность к мобильному — около **40%**. База есть: `viewport` прописан корректно, во многих местах используются адаптивные префиксы (`sm:`/`md:`/`lg:`), у всех кабинетов уже сделана нижняя таб-навигация (`fixed bottom-0 … lg:hidden`), а таблицы обёрнуты в `overflow-x-auto`. Но три вещи ломают реальный мобильный опыт: **экран входа**, **широкие таблицы** и **тесная нижняя навигация**. Логин — самое слабое место и должен идти первым.

## Что уже хорошо

`index.html` содержит корректный `<meta name="viewport">`. Большинство grid-раскладок адаптивны (`grid-cols-1 md:grid-cols-2 …`). График у преподавателя использует `ResponsiveContainer` (TeacherWorkspace.tsx:709). Все 6 таблиц обёрнуты в `overflow-x-auto`. Модалки используют `fixed inset-0 … p-4`. У ParentWorkspace и TeacherWorkspace переходы flex-col→flex-row сделаны аккуратно.

## Критичные проблемы (P0)

### 1. Экран входа практически неработоспособен на телефоне

Логин построен как жёсткие две колонки и абсолютное позиционирование по процентам — на узком экране всё разъезжается.

- `App.tsx:1341` — левая колонка `w-[42%] … p-10 shrink-0`, правая `flex-1 … p-12` (1506). Нет переключения на одну колонку: нужно `flex-col` на мобильном, `md:flex-row`.
- Поля и кнопки формы спозиционированы абсолютно в процентах от фоновой картинки: email `left-[56.2%] top-[28.5%]` (1225), пароль (1236), показать пароль (1242), «запомнить» (1246), «забыли пароль» (1261), кнопка входа (1267). На мобильном попадают мимо.
- Интерактивные «хотспоты» с жёсткими `top-[…%] left-[…%]` (1384, 1410, 1436, 1462) и тултипы `w-52` (1400, 1426, 1452, 1478) — на 480px перекрывают экран.
- Загружены оба фона (`login_desktop_bg.png`, `login_mobile_bg.png`), но мобильный используется только в симуляторе (5600), а не на реальном мобильном логине (1211).
- Жёсткая типографика без брейкпойнтов: `text-[52px]` (1358), `text-[26px]` (1491). Декоративные блобы `w-[350px] h-[350px]` (1509) больше экрана.

### 2. Таблицы с фиксированной минимальной шириной

Обёртка `overflow-x-auto` есть, но минимальные ширины заставляют горизонтально скроллить почти всё; card-варианта для узких экранов нет.

- AdminEduErpWorkspace.tsx: `min-w-[940px]` (297), `min-w-[720px]` (600), `min-w-[860px]` (898), `min-w-[420px]` (1021).
- OwnerExecutiveWorkspace.tsx: `min-w-[980px]` (451), `min-w-[320px]` (837).
- App.tsx: четыре таблицы (платежи, аудит-лог, диспетчер и др.) — `overflow-x-auto` есть, но `text-xs` + 6–7 колонок нечитаемы на 480px.

Рекомендация: для ≤768px давать card-раскладку (строка = карточка «поле: значение»), а таблицу прятать `hidden md:table`.

### 3. Нижняя таб-навигация перегружена

Таб-бар есть у всех ролей, но колонок слишком много для 480px — подписи обрезаются, тач-цели крошечные.

- 5 табов: AdminEduErpWorkspace.tsx:194, BranchManagerWorkspace.tsx:171, OwnerExecutiveWorkspace.tsx:166, StudentArtistCabinet.tsx:135.
- 6 табов: ParentWorkspace.tsx:190 (`grid-cols-6 … px-1.5`), TeacherWorkspace.tsx:191 (`flex justify-around max-w-2xl`).

Рекомендация: максимум 4–5 видимых пунктов, остальное — в «Ещё»; иконка + короткая подпись; высота пункта ≥48px.

## Важные проблемы (P1)

### Верхний хедер и сайдбар (App.tsx)

- Селектор ролей — 7 кнопок в ряд без `flex-wrap` (1742), на мобильном даёт переполнение. Решение: горизонтальный скролл или dropdown.
- Блок статуса/темы/аватара скрыт `hidden xl:flex` (1769) — на телефоне пользователь не видит, кто он. Стоит вынести в мобильное меню.
- Десктоп-сайдбар `hidden lg:flex w-64` (1884) — ок, но мобильный drawer закрывает весь контент.
- Сайдбары кабинетов используют `w-76` (AdminEduErpWorkspace.tsx:111, OwnerExecutiveWorkspace.tsx:102) — **этого класса нет в Tailwind по умолчанию** (шкала идёт `w-72` → `w-80`), ширина может не применяться. Проверить и заменить на валидный размер.
- На планшете (768–1024px) сайдбары `lg:block` уже скрыты, но контент при этом во всю ширину без боковой навигации — для планшета лучше `xl:block` + компактный режим.

### Тач-цели меньше 44px

Кнопки ролей `px-2.5 py-1.5` + иконка 14px ≈ 26px (App.tsx:1751); переключатель темы и закрытие drawer ≈ 28–36px (1831, 1852); пункты сайдбара `py-2.5` ≈ 28px (1897+); чекбокс «запомнить» `w-4 h-4` (1632). Минимум для тача — 44×48px.

### Узкие grid на промежуточных ширинах

`grid-cols-3` без адаптации в MiniTable (AdminEduErpWorkspace.tsx:269) и StudentVideoStudio.tsx:119; `grid grid-cols-2 gap-3` с плотным текстом (OwnerExecutiveWorkspace.tsx:407); скачок `grid-cols-2 → lg:grid-cols-4` без `sm:`/`md:` (App.tsx:2143). На 600–900px смотрится тесно.

## Рекомендуемый порядок

Сначала логин (P0-1) — без него на телефоне в приложение не войти. Затем таблицы и навигация (P0-2, P0-3) — это основной ежедневный контент. Потом полировка хедера и тач-целей (P1). Каждый блок проверять на 375px, 414px, 768px, 1024px (DevTools device toolbar).
