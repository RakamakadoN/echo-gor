# ТЗ: Раздел «Документолог»

**Проект:** Эхо Гор · **Роль:** только Владелец · **Дата:** 2026-06-25

## 1. Цель

Единый раздел для работы с договорами компании: хранилище подписанных
документов по папкам-категориям + генератор новых договоров из шаблонов
с условной логикой. Закрывает две боли — «где лежит договор» и «истёк срок
и никто не заметил».

## 2. Скоуп

- Доступ: **только Владелец** (ролевой скоуп, как у Бухгалтерии).
- Не входит в MVP: подписание онлайн, ЭЦП, согласование/маршруты, версии
  одного договора. Закладываем структуру так, чтобы добавить позже.

## 3. Две функции в одном разделе

### 3.1. Хранилище

Таблица договоров; к каждой записи прикреплён **PDF-скан** подписанного
документа. Папки = категории (справочник, см. §5).

Поля записи:

- тип/категория (из справочника)
- контрагент (название)
- предмет договора (короткое описание)
- сумма + валюта
- дата начала
- дата окончания (nullable — бессрочные)
- автопролонгация (да/нет)
- статус: `draft` / `active` / `expired` / `terminated`
- файл скана (PDF, в Supabase Storage)
- комментарий
- created_at / updated_at

**Напоминания о сроках** — ключевая фича. Индикатор «истекает через N дней»
на дашборде Владельца + в самом разделе. Порог N настраивается (дефолт 30).
Бессрочные и расторгнутые в напоминания не попадают.

### 3.2. Генератор договоров

Мастер: выбор типа → форма полей → **условная логика** подставляет пункты →
на выходе **docx**.

Условная логика (примеры переключателей):

- с НДС / без НДС
- предоплата / постоплата / частями
- с автопролонгацией / фиксированный срок
- с актом приёмки / без

Сгенерированный договор **сразу заводится записью в хранилище** со статусом
`draft`. Жизненный цикл документа:

> генератор (docx) → правки руками → печать → подпись → скан (PDF) →
> загрузка в ту же запись → статус `active`

Формат: **генерация в docx** (Владелец дорабатывает перед подписанием),
**хранение скана в PDF** (неизменяемый подписанный экземпляр).

## 4. Категории (стартовый набор)

- Аренда (залы, помещения)
- Услуги — уборка
- Услуги — вывоз мусора
- Подрядчики / поставщики
- Прочее

Хранятся в `settings_lists` (новый список `document_categories`), чтобы
добавлять типы без миграции и редактировать на вкладке «Настройки».

## 5. Данные (миграция 027)

Новая миграция `027_documents.sql`:

- таблица `documents` — поля из §3.1
- таблица `document_templates` — шаблоны генератора (тип, тело docx-шаблона
  с плейсхолдерами, схема полей формы + правила условной логики в JSON)
- сид: 5 категорий в `settings_lists`, 1–2 базовых шаблона (аренда, услуги)
- bucket в Supabase Storage для PDF-сканов и сгенерированных docx

Storage: учесть прод-дрейф миграций — миграцию пишем идемпотентно
(`if not exists`), применяем и локально, и на проде.

## 6. API (`/api/mvp/documents`)

- `GET  /api/mvp/documents` — список с фильтрами (категория, статус, срок)
- `POST /api/mvp/documents` — создать запись
- `PATCH /api/mvp/documents/:id` — изменить (в т.ч. статус)
- `DELETE /api/mvp/documents/:id` — удалить (с подтверждением)
- `POST /api/mvp/documents/:id/upload` — загрузка PDF-скана
- `GET  /api/mvp/documents/templates` — список шаблонов
- `POST /api/mvp/documents/generate` — генерация docx из шаблона + полей,
  создаёт запись `draft` и возвращает файл

Все write-роуты — в try/catch (по уроку прод-базы), ролевая проверка
владельца, mock-фолбэк для демо/E2E.

## 7. UI (`DocumentologistView.tsx`)

- Левая колонка: папки-категории + счётчики.
- Центр: таблица договоров (контрагент, предмет, сумма, срок, статус),
  фильтры по статусу и сроку, поиск.
- Бейдж «истекает через N дней» жёлтым, «истёк» красным.
- Кнопка **«Создать договор»** → мастер-генератор (шаги: тип → поля →
  переключатели → предпросмотр → скачать docx + создать запись).
- Карточка договора: поля + кнопка загрузки PDF-скана + смена статуса.
- Точка входа — пункт меню Владельца рядом с «Бухгалтерией».
- Блок-индикатор на дашборде Владельца: «Договоры на контроле» (истекающие).

## 8. Этапы

1. Миграция 027 + сид категорий и шаблонов + Storage bucket.
2. API documents (CRUD + upload), ролевой скоуп, mock-фолбэк.
3. UI хранилища (папки, таблица, фильтры, карточка, загрузка скана).
4. Напоминания о сроках + блок на дашборде Владельца.
5. Генератор docx (шаблонизатор + условная логика + автозапись в хранилище).
6. Вкладка «Настройки»: управление категориями и шаблонами.
